# exo A minimal substrate for building **durable** agents. > Exo separates the trusted infrastructure an agent needs — identity, history, secrets, sandboxes — from the policy code above it. The substrate stays put; the executor on top can be killed, swapped, or rewound without losing state. `rust core` · `typescript harness` · `codex` · `cursor sdk` · `claude code` · `mit` --- ## architecture

> log accumulates — independent of the executor executor appends events to the durable loglog accumulates — independent of the executorswap executor or rewind — substrate persistsfork a new branch from that event --- ## why Most agent harnesses conflate trusted infrastructure with agent-specific implementation — prompts, memory compaction, tool dispatch — making reuse, recovery, isolation, and self-modification hard. Exo pushes the minimum into a protected substrate so safety is enforced once, and leaves everything semantic to the executor above. The executor can be killed at any moment without taking the agent with it. - **fork & rewind** — Resume from any past event with secrets, sandboxes, and history intact. - **swap** — Run the same agent under Codex, the Cursor SDK, Claude Code, or your own executor — without rebuilding state. - **self-modify** — Agents can rewrite themselves (tools, prompts, even their own loop) while the substrate isolates the things that matter. - **durable** — Event log is source of truth. Everything is reconstructible from it. --- ## quick start Install Rust and pnpm, then: ```bash # build the cli $ cargo build -p exo # register a model $ exo secret set openai --env OPENAI_API_KEY $ exo model register gpt-5.5 --secret openai # drop into a repl $ exo repl ``` Or back Codex, the Cursor SDK, or Claude Code with a durable exoharness session: ```bash $ exo --harness typescript agent create \ --module examples/typescript/codex.ts \ --model gpt-5.5 \ "Coding Agent" # resume, fork, or swap at any time $ exo session resume <id> ``` --- ## repo - `crates/` — Rust workspace · CLI, exoharness substrate, executors - `typescript/` — Harness runtime, model-runtime helpers - `examples/typescript/` — Runnable Codex, Claude Code, Cursor harnesses - `containers/` — Sandbox images - `spec/` — Core architecture and terminology --- ## status Early software.
